## 一、项目定位

**银龄智盾 · 认知疫苗**是一套面向长辈的**反诈认知训练 + 智能陪伴对话**演示系统，典型场景为展览、课题演示或内部试用。

| 板块 | 功能概要 |
|------|----------|
| **找小孙聊天** | 文本对话；可选语音播报与「语音电话」（浏览器语音识别）。回复由 **DeepSeek**（OpenAI 兼容接口）**流式**生成；朗读经后端 **`/api/tts`**（优先 CosyVoice，否则 Edge TTS）。 |
| **防骗训练** | SQLite 题库、剧本与计分；支持读题（浏览器 `speechSynthesis`）。 |
| **我的报告** | 根据训练结果生成认知安全报告（雷达图、六维说明）；理论口径与《防诈意识指标及理论支撑》对齐（如满分合计 105 分）；支持导出 PDF。 |

## 三、环境变量（`backend/.env`）

在 **`backend` 目录**下放置 `.env`（与 `main.py` 同目录，供 `load_dotenv()` 加载）。

```env
# DeepSeek（未配置时陪伴聊天不可用）
DEEPSEEK_API_KEY=你的密钥
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

# 可选：陪伴模式温度，默认约 0.35
# DEEPSEEK_COMPANION_TEMPERATURE=0.35

# Edge TTS 音色（edge-tts 走微软云端，需联网）
EDGE_TTS_VOICE=zh-CN-XiaoxiaoNeural

# 可选：CosyVoice（本地/内网，配置后优先于 edge-tts）
# 需先单独启动 CosyVoice 官方 FastAPI，例如：
#   cd CosyVoice-main/runtime/python/fastapi
#   python server.py --port 50000 --model_dir <模型目录或 ModelScope ID>
# COSYVOICE_URL=http://127.0.0.1:50000
# COSYVOICE_SPK_ID=中文女
# 若听感语速异常，与模型 cosyvoice.yaml 中 sample_rate 对齐，例如：
# COSYVOICE_SAMPLE_RATE=22050
```

**安全**：密钥不要提交 Git；生产环境用环境变量或密钥管理服务；交接密钥用安全渠道。

## 四、本地启动

### 方式 A：命令行（推荐排错）

**1）后端**

```bash
cd backend
pip install -r requirements.txt
python main.py
```

浏览器访问：`http://localhost:8000/api/health`，应返回 `ok`、`model`、`shanghai_now`（北京时间）等。

**2）前端**

```bash
cd frontend
npm install
npm run dev
```

浏览器访问：**以终端打印的 Local 地址为准**（默认常为 `http://localhost:3000`，见 `vite.config.js`）。

### 方式 B：`start.bat`（Windows）

双击项目根目录 `start.bat`，会分别启动后端与前端。**端口以两个窗口内实际输出为准**。

### 前后端地址

- 前端通过 `frontend/src/apiBase.js` 解析 API 根地址：  
  - 开发环境默认 `http://127.0.0.1:8000`；  
  - 可在 **`frontend/.env`** 中设置 `VITE_API_BASE` 指向你的后端（生产构建同理）。  
- 若部署域名与端口变化，**改 `VITE_API_BASE` 后重新构建前端**，勿只改一处硬编码。

## 五、后端接口摘要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 服务、模型、北京时间自检 |
| POST | `/chat/companion` | 陪伴模式，流式文本（SSE/流式 body，与前端解析一致） |
| POST | `/api/tts` | 请求体 JSON `{"text":"..."}`；CosyVoice 成功时为 **WAV**，否则 Edge 为 **MP3** 流 |
| GET | `/api/scenarios` 等 | 防骗剧本与题目（SQLite） |

历史上另有 `/chat` 等演示接口，**是否与当前前端一致请以实际 `fetch` 调用为准**。

## 六、业务与实现要点（接手必读）

### 6.1 日期与「星期几」

模型不自带实时日历。`llm_service.py` 在陪伴对话的系统提示中注入 **Asia/Shanghai** 的当前时间与星期。若服务器时区或容器时间错误，「今天星期几」会不准，部署时注意时区配置。

### 6.2 朗读与浏览器自动播放策略

- 浏览器可能默认禁止**未由用户手势触发**的音频；首次有声播放通常需用户**点击页面**或通过首页进入聊天等操作完成「解锁」。  
- 朗读优先请求后端 TTS；失败时回退浏览器 `speechSynthesis`。  
- 展示与朗读前会对文本做**去括号旁白**等处理（与提示词约束一致）。

### 6.3 语音电话（语音识别）

- 使用 **Web Speech API**（`SpeechRecognition` / `webkitSpeechRecognition`），**Chrome / Edge** 体验较稳定；中文识别多依赖**在线服务**，需**能访问对应云端**的网络环境。  
- **说完一句**后一般以**最终结果**提交发送；单字杂音会过滤，单个汉字允许作为有效输入。  
- 为避免扬声器回声被当成用户说话，**小花朗读时会暂停麦克风**，读完后再恢复（详见 `ChatView.vue` 当前实现）。

### 6.4 报告与 PDF

- 六维指标与满分口径与《防诈意识指标及理论支撑》一致；页面含文档来源说明。  
- 训练相关得分可能使用 `localStorage`（如 `zhidun_final_score`），报告页读取并用于雷达图等展示。

## 七、常见问题

| 现象 | 建议排查 |
|------|----------|
| 聊天无回复或超时 | `DEEPSEEK_API_KEY`、网络、后端 8000 是否监听；前端请求超时时间。 |
| 有字没声音 | 后端 `/api/tts` 是否通；是否已有点击等交互解锁播放；系统音量。 |
| 前端连不上 API | `apiBase.js` / `VITE_API_BASE` 是否与后端一致；CORS；勿用 `file://` 直接打开构建产物测试跨域。 |
| 语音电话无反应 | 浏览器是否 Chrome/Edge；麦克风权限；网络（语音识别云端）；看页面是否提示网络类错误。 |
| 端口被占用 | 修改 `vite.config.js` 或释放进程后重启。 |
| 题库 / 训练异常 | `zhidun.db` 是否存在；是否需执行 `init_db.py` 等（以仓库脚本说明为准）。 |
