# 智盾·认知疫苗系统 — 项目交接说明

本文档供后续接手开发、部署与运维的同事使用，说明项目定位、架构、启动方式、配置项与常见问题。

---

## 一、项目是什么

**智盾·认知疫苗**是一套面向老年人的**反诈认知训练与陪伴对话**演示系统，包含：

| 模块 | 说明 |
|------|------|
| **找小孙聊天** | 与「小孙」文本/语音对话；回复由 **DeepSeek API**（OpenAI 兼容协议）流式生成；可选 **神经网络中文朗读**（Edge TTS，经后端 `/api/tts`）。 |
| **防骗训练** | 从 SQLite 题库拉取剧本与题目，答题影响得分；支持读题（浏览器语音合成）。 |
| **我的报告** | 根据训练得分生成「认知安全」报告：雷达图、六维说明；内容与《防诈意识指标及理论支撑》对齐（满分合计 105 分等），可导出 PDF。 |

## 二、技术栈与目录结构

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3、Vite、ECharts、html2pdf.js；适老大字号 UI。开发服务默认端口 **3000**（见 `frontend/vite.config.js`）。 |
| 后端 | Python 3、**FastAPI**、Uvicorn；SQLite（`backend/zhidun.db`）。默认监听 **0.0.0.0:8000**。 |
| 大模型 | **DeepSeek**（`openai` 官方 SDK + 自定义 `base_url`）。 |
| 朗读 | **edge-tts**（微软神经网络中文语音，需联网）。 |

```
zhidun-project/
├── backend/
│   ├── main.py              # FastAPI 入口：聊天流、TTS、题库、健康检查等
│   ├── llm_service.py       # DeepSeek 流式对话、陪伴/攻击/卫士提示词、北京时间注入
│   ├── init_db.py / seed_scenarios.py  # 数据库初始化与种子（若使用）
│   ├── requirements.txt
│   ├── .env                 # 密钥与模型（勿提交仓库）
│   └── zhidun.db            # SQLite
├── frontend/
│   ├── src/
│   │   ├── App.vue          # 三 Tab：聊天 / 训练 / 报告
│   │   └── components/
│   │       ├── ChatView.vue    # 聊天、语音电话、播报、打断逻辑
│   │       ├── TrainingView.vue
│   │       └── ReportView.vue    # 报告与 PDF、六维理论摘要
│   ├── vite.config.js
│   └── package.json
├── start.bat                # Windows 一键启后端+前端（注意前端端口以 Vite 配置为准）
├── README.md
└── 交接说明.md              # 本文件
```

## 三、环境变量（`backend/.env`）

在 **`backend` 目录下**放置 `.env`（与 `main.py` 同目录，便于 `load_dotenv()` 加载）。示例：

```env
# DeepSeek（必填，否则陪伴聊天会提示未配置）
DEEPSEEK_API_KEY=你的密钥
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

# 可选：陪伴模式温度，默认约 0.35
# DEEPSEEK_COMPANION_TEMPERATURE=0.35

# 朗读（edge-tts，需联网）
EDGE_TTS_VOICE=zh-CN-XiaoxiaoNeural
```

**安全：** 密钥勿写入版本库；交接时当面或安全通道传递，生产环境用环境变量或密钥管理服务。

## 四、本地启动

### 方式 A：命令行（推荐排查问题）

1. **后端**

```bash
cd backend
pip install -r requirements.txt
python main.py
```

浏览器访问：`http://localhost:8000/api/health` 应返回 `ok`、`model`、`shanghai_now`（北京时间）等。

2. **前端**

```bash
cd frontend
npm install
npm run dev
```

浏览器访问：**`http://localhost:3000`**（以 `vite.config.js` 为准；若改为 5173 则以终端输出为准）。

### 方式 B：`start.bat`

双击根目录 `start.bat` 会打开两个窗口分别跑后端与前端。脚本里若写「5173」而实际为 3000，**以 Vite 控制台打印的 Local 地址为准**。

### 前后端地址约定

- 前端内对 API 多写死为 **`http://localhost:8000`**（见 `ChatView.vue` 等）。若部署到不同机器/域名，需改前端 `API_URL` 或改为环境变量构建。

## 五、核心接口一览（后端）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 服务与模型、北京时间自检 |
| POST | `/chat/companion` | 流式文本（DeepSeek 陪伴模式） |
| POST | `/api/tts` | 朗读：请求体 JSON `{"text":"..."}`，返回 MP3 流 |
| GET | `/api/scenarios` 等 | 防骗剧本与题目（SQLite） |
| GET | `/api/training/...` | 题目详情 |

另有历史演示用 `/chat` 等，与当前主流程前端可能不一致，以实际前端调用为准。

## 六、业务逻辑要点（接手必知）

### 6.1 聊天与「今天星期几」

模型本身不知道实时日期。`llm_service.py` 在**每次**陪伴对话的系统提示中注入 **Asia/Shanghai** 当前时间与星期，并约束回答日期类问题须与此一致。服务器时区错误会导致星期错误，部署时注意机器时间或使用容器时区。

### 6.2 朗读与浏览器策略

- 朗读默认**开启**；用户需**点击页面**或「语音播报」关→开一次，浏览器才允许播放（自动播放策略）。
- 朗读走服务端 **edge-tts**，失败时回退浏览器 `speechSynthesis`。
- 文本会先去掉常见括号旁白再显示与朗读（与提示词「禁止括号旁白」配合）。

### 6.3 语音电话与打断

- 「语音电话」使用 Web Speech API 识别；**说完一句**后识别结果才会发送。
- 朗读时若开启语音电话，设计上支持**用户再开口打断**当前朗读（具体以 `ChatView.vue` 最新逻辑为准）。

### 6.4 报告与 PDF

- 六维指标与满分与《防诈意识指标及理论支撑》文档对齐；页面有文档来源说明。
- 训练得分存 `localStorage`（如 `zhidun_final_score`），报告页读取并折算雷达图。

## 七、常见问题

| 现象 | 排查方向 |
|------|----------|
| 聊天无回复或超时 | 检查 `DEEPSEEK_API_KEY`、网络、8000 是否启动；前端有请求超时时间限制。 |
| 有字没声音 | 后端是否可访问 `/api/tts`；是否已用户点击解锁播放；系统音量。 |
| 前端连不上后端 | CORS 已放开；确认 API 地址为 8000 且未跨域混用 file://。 |
| 端口冲突 | 修改 `vite.config.js` 或结束占用进程后重启。 |
| 数据库异常 | 检查 `backend/zhidun.db` 是否存在；必要时运行 `init_db.py`（以项目脚本为准）。 |
